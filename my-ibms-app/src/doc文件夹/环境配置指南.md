# IBMS 工控管理系统 - 环境配置指南

> 从 GitHub 下载后打不开？按下面步骤一步步来就好。

---

## 1. 前置条件：检查 Node.js 版本

这个项目使用 **Vite 8 + React 19**，对 Node.js 版本有要求。

### 检查当前版本

打开终端（PowerShell / CMD / Git Bash 都可以），输入：

```bash
node -v
```

**要求：Node.js >= 18.0.0**（推荐 20.x 或 22.x LTS 版本）

### 如果版本太低 / 没有装 Node.js

去官网下载安装 LTS 版本：https://nodejs.org/

安装完成后重新打开终端，再次运行 `node -v` 确认版本正确。

> ⚠️ **常见踩坑**：安装完 Node.js 后，之前打开的终端窗口不会自动识别新版本，需要关闭终端重新打开。

---

## 2. 安装依赖

在项目根目录下运行：

```bash
npm install
```

这个命令会读取 `package.json`，自动安装所有需要的依赖包，包括：
- React 19
- Ant Design 6
- React Router 7
- Vite 8
- 以及其他开发依赖

> 如果 `npm install` 很慢，可以先换淘宝镜像：
> ```bash
> npm config set registry https://registry.npmmirror.com
> ```
> 然后再 `npm install`。

---

## 3. 启动开发服务器

依赖安装完成后：

```bash
npm run dev
```

终端会输出类似以下信息：

```
  VITE v8.x.x  ready in xxx ms

  ➜  Local:   http://localhost:5173/
  ➜  Network: http://192.168.x.x:5173/
```

在浏览器打开 `http://localhost:5173/` 就能看到系统了。

## 5. 常见问题排查

### Q1：运行 `npm install` 报错一大堆

**可能原因**：Node.js 版本太低。

**解决**：升级 Node.js 到 20.x 或 22.x LTS 版本，然后删除 `node_modules` 文件夹和 `package-lock.json`，重新 `npm install`。

```bash
# 删除后重装
rm -rf node_modules package-lock.json   # Mac/Linux/Git Bash
# 或 PowerShell：
# Remove-Item -Recurse -Force node_modules, package-lock.json

npm install
```

### Q2：`npm run dev` 后浏览器白屏 / 报错

**可能原因**：依赖没装全。

**解决**：删除 `node_modules` 文件夹，重新 `npm install`。

### Q3：端口 5173 被占用

Vite 会自动尝试下一个端口（5174、5175...），看终端输出确认实际端口。

也可以指定端口：

```bash
npm run dev -- --port 3000
```

### Q4：Windows PowerShell 报"无法加载文件...禁止运行脚本"

**解决**：以管理员身份打开 PowerShell，运行：

```powershell
Set-ExecutionPolicy RemoteSigned -Scope CurrentUser
```

输入 `Y` 确认，然后重新打开终端。

### Q5：启动后页面样式错乱

**可能原因**：浏览器缓存了旧版本。

**解决**：按 `Ctrl + Shift + R`（Mac: `Cmd + Shift + R`）强制刷新，或打开无痕窗口试试。
